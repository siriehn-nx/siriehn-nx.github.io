MovieChat论文学习笔记

Abstract

使用 integrating video foundation models 和 large language models 组合解决视频理解系统，客服了特定的视觉预处理任务。

但是现有的困难是只能处理总帧数不多的视频，对于长视频的（空间复杂度，时间复杂度，远程时间链接）还是尚待解决的问题。受到 Atkinson-Shiffrin memory model 的启发，作者设计了一种记忆方法，其中包括快速更新的短期记忆和稳定连续的长期记忆。他声称 Moviechat 在长视频理解方面有最好的性能。

Related Work

  • Video Foundation Models 视频基础模型（我前期看的论文有这个，但是后来这篇论文把它删了）
  • Multi-modal Large Language Models 多模态大语言模型
  • Long Video Understanding 长视频理解
  • Memory Models in Vision Tasks 视觉任务的记忆模型

MovieChat

创新处：减少了视觉冗余信息，提出了创新记忆模型。

  • frame-wise visual feature extractor 逐帧特征提取器

  • the short-term and long-term memory buffers 长期和短期视频处理器

  • the video projection layer 视频投影层（不管了，先往后看）

  • the Large Language Model 大语言模型

为了解决在 GPU 和 RAM 中存储大量帧数的问题，采取滑动窗口的方式处理短期记忆。

两种模式：

  • Breakpoint mode：处理特定帧/场景下的理解和答案
  • Global mode：处理全局视频理解

Video Features Extraction

不再使用传统的 Video based foundational model，而是用 image-based model去得到逐帧的特征，因为没有视频基础模型和文本很好地结合，并且这篇文献的记忆机制可以很好捕捉时间特征。

逐帧特征提取器是 ViT-G/14

输入是给定视频 \(v \in \mathbb {Z} ^ {T \times 3 \times H \times W}\) 这是一个长度为 \(T\) 的宽度为 \(W\) 高度为 \(H\) 的 \(\text{RGB}\) 序列。使用滑动窗口提取特征序列 \(B_n = \{x_i = \mathcal{V}(v_i) \vert i = 1, \ldots, C\}, n = 1, \ldots, \left\lceil \frac{T}{C}\right\rceil\)，这后面我在查一下 \(\mathcal V\) 是视频提取器 \(x_i \in \mathbb R ^ {N \times D}\)，\(N\) 是 tokens 的数量，\(D\) 是特征维度。

Short-Term Memory

通过把 visual tokens 存在缓冲区里面，用滑动窗口 \(K\) 次提取出来的视觉特征构建短期记忆 \(\mathcal S = \bigcup\limits_n B_n = \{x_i \vert i = 1 , \ldots , K\}, n = 1, \ldots, G, K = C\times G\)，短期记忆的更新策略就是队列式。

Long-Term Memory

前面提到由于 GPU 和 Memory Limitation 的限制，所以我们需要构建长期记忆。时间上相似的帧可能在内容上也具有相似性，所以这篇文章提出了一种合并时间上相邻内容上具有相似性的 tokens 将其变成 sparse memory 存到 Long-Term Meomory 中的算法。

定义相似度是 \(s = \frac{1}{N} \sum\limits_{j = 1} ^N cos(x_i ^ j , x_{i + 1} ^ j)\)，合并两个相邻 tokens 的方式是 ToMe https://arxiv.org/pdf/2210.09461.pdf。每次合并操作之后保留 \(R_L\) 帧（这是个参数），产生输出视频特征 \(v' \in \mathbb Z ^{R_L \times 3\times H \times W}\)，

执行这个算法直到 Short-Term 中的 tokens 达到合并操作设定的值.

Extend positional encoding：长期记忆的 tokens 数量如果过多，采取 the hierarchically decomposed positional encoding method 方法可以将绝对位置编码从 \(n\) 扩展到 \(n^2\)

创新：以前的方法使用整个视频的表示法，所以无法定位到某个时刻，这个模型使用了两种模式全局和断点。

  • Global mode：整个视频表示，使用 Long-Term Memory \(\mathcal L\) 表示整个视频
  • Breakpoint mode：理解视频里面的某个特定时刻，我们需要考虑 \(\mathcal{L, S}, x_t\) 里面的东西，直接连接这些项形成视频表示 \(V\)，经过 Q-former 和 a linear projection layer 之后送到 LLM 中，可以表示为 \(A = \mathcal{O}(Q, \mathcal P (V))\)，\(\mathcal P(V)\) 是视频表示 \(V\) 到文本曾投影，\(Q\) 是问题。

MovieChat-1K

为了测试性能，提出了一种新的 benchmark 叫做 MovieChat-1K，他包含 1K 视频剪辑和 14K 手动批注。

超过 90% 的视频显示的持续时间在 10K 到 12K 帧之间，而 14.6% 的视频超过 12K 帧。只有 8.6% 的视频持续时间小于 10K 帧。

这里面测试出来对于短视频的 MovieChat 也遥遥领先。

Experiments

对于以前的方法和 Moviechat 进行了评估测试，还进行了 ablation studies.

这一部分没有仔细看

实验结果和分析： 该论文通过实验验证了提出的 MovieChat 模型在长视频理解任务上的性能。实验中，作者使用了基于图像的模型作为视觉特征提取器，并采用了短期记忆和长期记忆的记忆机制。实验结果表明，MovieChat 在长视频理解任务上取得了最先进的性能。

Limitation

  • 由预先训练的段视频模型决定感知能力
  • 对于长视频中事件持续时间的估计能力不足。
